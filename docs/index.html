§01 The problem

Server
archaeology

Agentic coding makes it very easy to create useful software faster than we create discipline around it. That is the trap.

rip.sh2019 · author: you, allegedly
rip_v2_FINAL.sh2020 · calls rip.sh, sometimes
fix_bluetooth.sh2021 · cron @reboot
backup.sh.old2022 · does not back up
crontab -eundated · author unknown
/usr/local/bin/helperload-bearing · do not touch

FIG. 01 — excavation of /usr/local/bin. Every layer was “just a quick script.”

A shell script becomes a service. A service becomes infrastructure. Infrastructure becomes the thing you forgot was holding up the shed.

The fun part is the idea — the protocol dance, the DVD drive that rips on insert, the speaker that behaves. The unfun part is the part that matters six months later: install layout, idempotence, logs, service ownership, config preservation, health checks, rollback, update hygiene, documentation that is not just vibes in Markdown.

Muster exists to offload that boring work to the agent — but in a constrained way. Not “make me a script.” More like:

# the whole deal, in one prompt
Here is the architecture.
Produce the operational harness.
Pass muster.

That constraint is the point. Agentic coding should make small tools more productionizable, not more haunted.

§02 The shape · systemd owns lifecycle

One boring,
load-bearing shape

Every Muster repo installs into the same skeleton. The exact service changes by project — the operational shape does not. Shown here: example/dvd-ingester, a real appliance from the repo.

/etc/dvd-ingester/ # config · survives updates
dvd-ingester.env
/opt/dvd-ingester/
releases/
1.0.2/ # immutable
1.0.3/ # immutable
current -> releases/1.0.3 # the only moving part
/var/lib/dvd-ingester/ # state · failure markers
90-dvd-ingester.rulesudev: matches the drive, asks systemd
dvd-rip@sr0.serviceone bounded rip job
dvd-publish-one.timeratomic publish drain
dvd-ingester-doctor.timerhealth inspection
dvd-ingester-update.timerupdate polling

systemd owns lifecycle. udev is allowed one verb: ask systemd to start a unit. Long work never runs inside a udev rule, a login shell, or a crontab line someone will discover in 2031.

Releases are immutable. Code installs under /opt/<project>/releases/<version>/ and units call scripts through current. Updating a service is moving one symlink. Un-updating a service is moving it back.

Health is a program, not a feeling. doctor.sh checks layout, units, capabilities, and the important failure paths. The updater refuses to keep any release the doctor won’t certify.

Below is the part that keeps 11 p.m. SSH sessions out of your future: the update rail, modeled live. Ship a good release. Then ship a bad one — the bad one is the whole point.

§03 The pattern library · MPL

A vocabulary of
operational shapes

Before a Muster repo invents anything bespoke, it names its shape in the Muster Pattern Library — 35 patterns across three tech tiers. Tech I atoms are single-concern systemd shapes. Tech II composes them into appliance behaviors. Tech III assembles whole machines. Rarity runs common → rare → mythic, which is where the sheaves and holonomy live. MRL is the Muster Readiness Level, 0–9. Select a composed pattern and pull its rigging apart — every node is real, from the manifests.

§06 The contract · MUSTER.md is the law book

The inspection
sheet

The law book now includes the shared host inspector and object graph; these are the clauses that hold the roof up. A repo that can’t stamp every row doesn’t ship.

01systemd owns service lifecycle.If it isn’t a unit, it isn’t real. Restart policy is not a suggestion.stamped
02systemd timers own scheduled work.Health checks and update polling are timers — not crontab folklore.stamped
03Config lives under /etc/<project>/.And survives every install, update, and uninstall short of --purge.stamped
04Code lives under /opt/<project>/releases/<version>/.Immutable and versioned. current is a symlink, and it is the only moving part.stamped
05Installers are idempotent.Run twice, converge twice, exit 0. Boredom is the acceptance test.stamped
06Updates verify SHA256, then earn their keep.Stage, flip, doctor. Fail the doctor and the symlink flips right back.stamped
07doctor.sh proves health.A service that cannot demonstrate it is healthy is presumed haunted.stamped
08The README self-certifies.A table of claims with evidence — and agent instructions that make Codex prove the repo is not merely plausible.stamped
09Every implementation explains itself through one graph.Globally addressable components, recursive health, exact release locks, and structured evidence feed every view.stamped
